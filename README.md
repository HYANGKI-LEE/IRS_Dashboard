# IRS 브로커 채팅 대시보드

카카오톡 채팅 내보내기(.txt)에서 IRS(금리스왑) 호가 데이터를 파싱해서
Bid/Offer 현황, 만기별 분포, 시간대별 활동량 등을 보여주는 Streamlit 대시보드.

## 새 채팅 파일 추가하는 방법

1. GitHub 저장소(`HYANGKI-LEE/IRS_Dashboard`) 접속
2. `data/` 폴더로 들어가서 **Add file → Upload files**
3. `.txt` 파일 업로드 후 커밋 (터미널 불필요)
4. Streamlit Cloud가 push를 감지해서 1분 내로 자동 재배포됨

파일명은 `회사명_이름.txt` 형식을 따르는 걸 권장 (대시보드에서 "회사(파일)" 필터로 씀).

## 텍스트 형식 규칙

- 날짜 헤더: `[2026년 7월 8일 수요일]`
- 메시지: `이름 (HH:MM:SS) : 내용`
- 한 메시지가 여러 줄이면 첫 줄에만 `이름 (시간) :`가 붙고, 나머지 줄(빈 줄 포함)은
  전부 같은 메시지로 취급됨 — 새 메시지 줄이나 날짜 헤더가 나오기 전까지는 안 끊김.

## 새로운 은어/표현 추가하기

파서가 못 알아듣는 표현은 대시보드의 **"🔍 미분류 감사"** 탭에서 raw_text 그대로 보임.
새 표현을 발견하면 `parser/taxonomy.py`에 키워드/정규식만 추가하면 됨 (다른 파일은 안 건드려도 됨):

- 방향/액션(Bid/Offer/거래 등) → `SIDE_ACTION_RULES`
- 청산/venue 태그(CCP/ND/청산 등) → `CLEARING_TAG_PATTERNS`
- 상품 구분(나비/베이시스 등) → `INSTRUMENT_RULES`
- 만기 단위 → `_UNIT`

## 알려진 파싱 한계 (의도된 동작, 버그 아님)

- `"4년 95.5/95.25"`처럼 비드/오퍼 단어 없이 가격만 있는 줄은 방향/액션이 "미분류"로
  표시됨 (원문에 방향이 안 적혀 있으므로) — 만기/가격 필드는 정상적으로 채워짐.
- 시황 요약 블록(`<<Market trades...>>` 같은)은 통째로 한 메시지로 처리되어 하위 줄들이
  전부 "미분류"로 떨어짐. 에러는 안 나지만 미분류 비율을 올리는 원인이라 감안하고 볼 것.
- `"6*9*12 나비"`처럼 만기 단위(년/개월)를 생략하고 바로 "나비"가 붙는 경우, 만기 숫자는
  잡히지만 단위는 비어있음.
- 양방향 가격(`95.5/95.25`)은 어느 쪽이 bid이고 어느 쪽이 offer인지 자동으로 판단하지
  않음 — `rate_1`/`rate_2`로 순서만 보존되므로 raw_text와 같이 보고 판단할 것.
- 한 줄에 가격이 두 개 겹친 복합 표현(예: `"4년 92 기븐 /91.75비드 청산"`)은 하나만
  구조화되어 잡힐 수 있음 — raw_text에는 항상 원문 전체가 남아있어 데이터 손실은 아님.

모든 원문은 `raw_text` 컬럼에 항상 그대로 보존되므로, 테이블/CSV에서 파싱 결과와
원문을 나란히 대조할 수 있음.

## 로컬에서 실행하기

```
pip install -r requirements.txt
streamlit run app.py
```

## 테스트

```
pip install -r requirements.txt
python -m pytest tests/ -v
```

## 배포 (Streamlit Community Cloud)

1. share.streamlit.io 접속 → GitHub 계정으로 로그인
2. 저장소 접근 권한 승인
3. New app → 저장소 `HYANGKI-LEE/IRS_Dashboard`, 브랜치 `main`, 메인 파일 `app.py` → Deploy
4. 발급된 `*.streamlit.app` 링크를 팀원과 공유 (로그인 불필요, 링크만 있으면 접속 가능)

⚠️ 이 저장소와 배포된 대시보드는 **누구나 링크로 접속 가능한 공개 상태**로 운영하기로
결정됨 — 브로커 실명, 호가, 거래 데이터가 노출된다는 점을 인지하고 진행할 것.
